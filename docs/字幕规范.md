# 字幕规范

## 基本原则

字幕是阅读层，不是总结层。

中文字幕默认贴近口播原话，只做必要修正：

- 错别字
- 专有名词
- 标点
- 断句
- 明显无意义口癖
- 过长行拆分

不要把口播改写成课程提纲、PPT 文案或编辑总结。

## 双语字幕

默认烧录中英双语：

- 中文主行在上
- 英文辅助行在下
- 中文更大、更重
- 英文更小、更短

推荐样式：

- 中文：`56-60px`，粗体，白色，带轻微阴影或描边
- 英文：`36-40px`，中粗，灰白色，透明度略低
- 行距：`8-14px`
- 位置：下中部，避开嘴和底部 UI

如果中文需要两行，优先缩短英文或省略该条英文，不要做成四行字幕块。

## 分句

每条中文字幕尽量：

- 8-18 个汉字
- 按语义和呼吸切分
- 不拆词
- 不留下单字或两字孤行
- 不用过长两行块压迫画面

## 英文翻译

英文服务观看节奏，不追求逐字翻译。

规则：

- 保留人名、产品名、工具名原文
- 简洁自然
- 一行优先
- 不和中文争视觉注意力

## 文件命名

```text
captions-zh-polished.srt
captions-en-polished.srt
```

如果存在原始转写：

```text
raw/source.srt
srt_work/source.whisper.json
srt_work/source.audio.16k.wav
```

## QA

渲染前后都要检查：

- 字幕和声音是否明显对齐
- 有没有漏掉关键句
- 有没有把整段话压成一句总结
- 有没有专名错误
- 有没有盖住嘴
- 有没有进入底部危险区
- 英文是否过长溢出

# Codex 工作流

## 一句话启动

```text
请按 AGENTS.md 完成 episodes/EPXX_标题 的视频日记剪辑。先盘点素材和规划；如果原片不干净，先做粗剪母版给我确认；确认后再进入正式剪辑、渲染和 QA。
```

## 阶段 1：确认素材

Codex 应先列出：

- 原片路径
- 字幕路径
- 截图/录屏/图片列表
- 口播稿或主题说明
- 原片是否已经清洗
- 缺失项
- 隐私风险

如果缺口会影响成片，先停在规划阶段，让用户补素材。

## 阶段 2：生成或校对字幕

优先顺序：

1. 使用用户提供的 SRT。
2. 使用用户提供的转写稿重新整理 SRT。
3. 用本地 Whisper 转写生成 SRT。

保存文件：

```text
captions-zh-polished.srt
captions-en-polished.srt
```

## 阶段 3：粗剪母版

如果原片不干净，先做口播清洗母版。

适合粗剪删除的内容：

- 口误、错名字、事实 slip
- 废弃开头、重复起句、重复解释
- 长停顿、空拍、设备操作、死空气
- 破坏理解且不承载个性的局部填充

不应该删除的内容：

- 完整论证转折
- 真实案例和具体名字
- 有用的停顿和情绪
- 视频日记本身的自然节奏

输出文件：

```text
粗剪母版/EPXX_口播清洗母版_v1.mp4
粗剪母版/EPXX_口播清洗母版_v1.srt
粗剪母版/EPXX_口播粗剪删减表.md
```

删减表至少包含：

| 原始时间段 | 删除原因 | 处理判断 |
| --- | --- | --- |
| 00:00:00-00:00:00 | 口误/重复/空拍 | 安全删除/需用户确认 |

粗剪完成后暂停，等用户确认。用户确认后，后续正式剪辑以清洗母版为源视频。

## 阶段 4：剪辑规划

在 `剪辑规划.md` 中补全：

- 核心主线
- 视觉节奏
- 证据素材使用表
- 卡片清单
- 字幕和安全区注意事项
- QA 时间点

## 阶段 5：制作工程

推荐工程结构：

```text
project/
├── index.html
├── DESIGN.md
├── package.json
├── assets/
├── renders/
└── qa/
```

`DESIGN.md` 记录：

- 原片规格
- 字幕策略
- 视觉覆盖层
- 证据素材使用
- 安全区处理
- 已知取舍

## 阶段 6：渲染

优先使用当前环境可用的稳定渲染方式。可以是：

- HyperFrames
- Remotion
- HTML/CSS/JS + Playwright/Chromium 截帧
- FFmpeg + 图片/字幕叠加

选择原则：稳定、可复现、能通过 QA。

## 阶段 7：QA

至少抽帧检查：

- 开头身份
- 第一张卡
- 每个证据素材
- 字幕密集段
- 口播者靠近镜头处
- 后半段关键转折
- 结尾身份

并用 `ffprobe` 或等价方式确认规格。

如果做过粗剪，还要确认：

- 原始视频和原始 SRT 没被覆盖。
- 清洗母版 MP4 可播放，规格正常。
- 清洗母版 SRT 和新时间线对齐。
- 删减表记录完整。
- 没有删掉真实内容转折或关键案例。

## 阶段 8：交付

最终回复格式：

```text
已完成：
- 成片：...
- 工程：...
- 粗剪母版：...
- 粗剪删减表：...
- 中文字幕：...
- 英文字幕：...
- QA：检查了开头/证据/字幕/结尾，规格为 ...
- 注意：...
```

## 常见中断

如果出现以下问题，不要硬编：

- 原片缺失
- 音频无法读取
- 字幕严重漂移
- 口误或错名字无法自然剪掉
- 证据素材无法识别
- 私密信息无法安全脱敏
- 本地缺少渲染依赖

先说明卡点和下一步最小解决方案。

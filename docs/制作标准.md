# 视频日记制作标准

## 目标

把真实口播剪成朴素、可信、有记录感的视频日记。剪辑的作用是降低理解成本，而不是把创作者包装成广告主。

## 基础规格

- 画幅：9:16 竖屏
- 推荐尺寸：`1080x1920`
- 推荐帧率：`30fps`
- 推荐编码：H.264 + AAC
- 主音频：口播原声
- 字幕：默认中英双语烧录

## 粗剪母版

如果原始口播还不干净，第一步不是做卡片和包装，而是先做一版干净的口播母版。

粗剪母版是原片和正式成片之间的确认关卡。它只解决“这段口播能不能作为正式剪辑源”的问题。

### 输入

- 原始口播视频
- 原始 SRT 或转写稿
- 用户标注的口误、错名字、重复段、必须保留段

### 输出

```text
粗剪母版/EPXX_口播清洗母版_v1.mp4
粗剪母版/EPXX_口播清洗母版_v1.srt
粗剪母版/EPXX_口播粗剪删减表.md
```

### 可以删除

- 口误、错名字、事实 slip
- 废弃开头、重复起句、重复解释
- 长停顿、空拍、设备操作、死空气
- 不影响个性和节奏的局部填充

### 不要删除

- 真实案例
- 关键判断
- 论证转折
- 有用的呼吸和情绪
- 创作者自然的表达节奏

### 硬规则

- 永远保留原始视频和原始 SRT。
- 不靠改字幕掩盖音频里的口误。
- 能自然剪掉就剪；不能自然剪掉就写进删减表，交给用户确认。
- 粗剪阶段不加截图、卡片、双语字幕样式、开头身份和结尾身份。
- 用户确认粗剪母版之后，才进入正式视觉包装。

## 安全区

以 `1080x1920` 为基准：

- 顶部危险区：`0-190px`，不放重要文字、日期、编号、logo。
- 开头/结尾身份区：从 `190-230px` 附近开始。
- 上方内容区：`220-720px`，适合简短概念卡、关键词卡。
- 横向截图/桌面录屏证据区：通常从 `560-620px` 左右开始，宽度 `92-96%`，底部停在字幕上方。
- 竖屏录屏/手机素材：接近全屏，只留窄边距。
- 字幕目标区：通常 `1380-1545px`，根据嘴部位置调整。
- 底部危险区：`1620-1920px`，不放字幕、CTA、关键信息。

## 视觉语言

- 保留原始口播画面质感，不加大面积奇怪滤镜。
- 卡片使用深色半透明底、白色主字、少量暖色强调。
- 字体用干净中文无衬线：系统字体即可。
- 卡片要少，且必须对应当前口播。
- 不用霓虹、HUD、复杂粒子、过度渐变、花哨转场。
- 不使用无意义装饰图标。

## 开头和结尾

开头可显示：

```text
视频日记
YYYY-MM-DD
DayXX
```

结尾可再次显示系列身份和一句自然收束。

规则：

- 开头保持 3-6 秒。
- 结尾保持 2-4 秒。
- 身份信息不得和内容卡叠在一起。
- 如果开头马上进入重点，宁可缩短身份，也不要遮挡内容。

## 证据优先

截图、网页、录屏、聊天记录、文件导出、产品图、现场视频都是证据。

证据画面规则：

- 需要看清时，放大成主视觉。
- 不要塞进小卡片里充当装饰。
- 横向证据用大宽屏面板。
- 竖向证据尽量接近全屏。
- 证据出现期间，可以把口播底图适度暗化或模糊。
- 可以盖住脸，但要盖得干净，不要露出半张脸或嘴。
- 叠加录屏默认静音，口播音频保持主轨。

## 卡片使用

卡片只用于降低理解负担：

- 定义一个概念
- 标出一个判断
- 提示一个转折
- 承接一个结构
- 简短说明证据素材

不要每句话都加卡。一个 3-5 分钟视频通常 6-10 个覆盖层已经够多。

## 后半段连续性

如果视频后半段仍在推进论证，不要完全空掉。可以用少量紧凑卡提示：

- 当前判断
- 新的转折
- 关键例子
- 最终落点

如果后半段只是自然收束，可以保持干净口播。

## 最终检查

发布前必须确认：

- 没有顶部安全区碰撞
- 字幕不盖嘴
- 字幕不进底部危险区
- 证据画面可看清
- 卡片和口播语义匹配
- 没有未脱敏隐私
- 结尾没有突兀断掉
- 如果原片不干净，已经先确认粗剪母版

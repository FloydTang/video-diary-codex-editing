# 视频日记 Codex 剪辑

一个给 Codex 使用的公开视频日记剪辑模板仓库。

它不是传统剪辑软件模板，而是一套可复用的工作协议：你把口播视频、字幕、截图、录屏和想法放进单期目录，Codex 按本仓库的规范完成脚本整理、字幕校对、画面规划、HyperFrames/HTML 组合、渲染和 QA。

## 适合什么内容

- 9:16 竖屏口播视频日记
- 创作者日更/周更工作记录
- 创业、产品、学习、项目复盘类短视频
- 需要在口播上叠加截图、录屏、网页、聊天记录、过程证明的内容
- 希望字幕、卡片、证据画面风格长期统一的系列内容

## 不适合什么内容

- 强包装广告片
- 高密度课程切片
- 依赖复杂转场和大量特效的商业宣传片
- 纯横屏长视频节目

## 快速开始

1. 克隆或复制本仓库。
2. 新建一期：

```bash
bash scripts/new_episode.sh EP01 我的第一条视频日记
```

3. 把素材放进生成的目录：

```text
episodes/EP01_我的第一条视频日记/
├── raw/              # 原始口播视频、原始字幕、原始录屏
├── assets/           # 截图、图片、网页导出、证明素材
├── 口播稿.md
├── 素材清单.md
└── 剪辑规划.md
```

4. 在 Codex 里打开本仓库，对 Codex 说：

```text
请按 AGENTS.md 和 docs/制作标准.md，完成 episodes/EP01_我的第一条视频日记 的视频日记剪辑。
先检查素材和规划，确认缺什么；如果素材足够，再生成字幕、剪辑项目、渲染成片并做 QA。
```

## 推荐素材

- 一条竖屏口播原片，优先 `1080x1920 / 30fps`
- 一份 SRT 字幕；没有也可以让 Codex 用本地 Whisper 生成
- 3-8 个关键截图或录屏
- 一份简短说明：这期想表达什么、哪些素材必须出现、哪些素材不要用

## 基本风格

- 真实口播优先，不做成广告片。
- 卡片少而准，只服务当前句子或段落。
- 截图和录屏是证据，不是装饰；该看清时就放大。
- 字幕是阅读层，不是总结层；默认中英双语烧录。
- 输出前必须做规格检查、抽帧检查和隐私检查。

## 公开使用提醒

这个仓库只提供通用流程和模板，不包含任何个人素材、品牌资产或真实项目文件。你自己的原片、字幕、截图、聊天记录和客户资料请放在 `episodes/` 下，并在发布或分享前按 `docs/隐私与公开前检查.md` 做脱敏。
